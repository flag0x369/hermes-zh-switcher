# Security Policy

## Reporting a Vulnerability

Please do not post vulnerabilities, tokens, local paths, profile names, account names, screenshots with private data, or sensitive logs in public issues.

Report security concerns privately by opening a GitHub security advisory for this repository. If advisories are unavailable, open a minimal public issue that says you have a security report to share, without including sensitive details.

## Supported Versions

Security fixes are handled on the default branch. Please use the latest published version or latest commit before reporting a suspected vulnerability.

## Scope

This project patches a local Hermes Desktop `app.asar` and should be reversible. Reports about accidental credential reads, unsafe writes, package contents, backup handling, GitHub Actions behavior, or leakage of local app data are in scope.
